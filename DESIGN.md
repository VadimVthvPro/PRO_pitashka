## 1. Visual Theme & Atmosphere

### Концепция: «Тёплый Бруталист»

Не «ещё один фитнес-сайт» с градиентами и неоновым зелёным. PROpitashka — это **тёплый, уверенный, немного дерзкий** продукт. Ассоциации: крафтовая кофейня, где считают макросы. Грубоватая типографика, честные цвета, живые текстуры.

**Философия:** Минимализм без стерильности. Каждый элемент несёт функцию. Пустое пространство — не роскошь, а дыхание. Асимметрия вместо идеальной центровки. Типографика как главный визуальный инструмент, а не иллюстрации.

**Плотность:** Средняя. Dashboard-экраны плотнее, формы ввода — просторнее. Контент дышит, но экран не пуст.

**Настроение:** Мотивирующее, но без крика. Как тренер, который верит в тебя, но не орёт.

### Impeccable AI Slop Test

Если человек посмотрит на этот сайт и скажет «это сделала нейросеть» — дизайн провалился. Проверочный список:
- Нет фиолетовых градиентов
- Нет cyan-on-dark
- Нет glassmorphism
- Нет карточек внутри карточек
- Нет glowing accents на тёмном фоне
- Нет gradient text
- Шрифт — не Inter, Roboto, Open Sans, Montserrat, Lato, Arial
- Не всё центрировано
- Нет «hero metric layout» (большое число + маленький лейбл + 3 стата)

## 3. Typography Rules

### Шрифты

| Роль | Шрифт | Fallback | Где используется |
|------|-------|----------|-----------------|
| **Display** | **Bricolage Grotesque** | Georgia, serif | Hero-заголовки, числа-метрики, названия секций |
| **Body** | **DM Sans** | -apple-system, BlinkMacSystemFont, sans-serif | Основной текст, формы, кнопки, навигация |
| **Mono** | **JetBrains Mono** | monospace | Числа в таблицах, калории, граммы, код |

**Почему эти шрифты:**

- **Bricolage Grotesque** (Google Fonts) — дизайнерский гротеск с характером. Широкие пропорции, нестандартные пластины. Мгновенно узнаваем, но отлично читается. Не «ещё один геометрический санс».
- **DM Sans** (Google Fonts) — геометрический, но с теплотой. Чуть мягче чем Inter, с более человечными пропорциями. Достаточно нейтрален для body, но не безликий.
- **JetBrains Mono** — для числовых данных. Табулярные цифры, лигатуры. Не для «технического вайба», а для выравнивания колонок с числами.

### Подключение шрифтов

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

```

Базовый размер: `1rem` = `16px`. Все размеры в `rem`.

| Токен | Размер | Вес | Line-height | Letter-spacing | Шрифт | Применение |
|-------|--------|-----|-------------|----------------|-------|------------|
| `display-xl` | `3.052rem` (48.8px) | 800 | 1.05 | `-0.03em` | Bricolage Grotesque | Hero-заголовок (один на страницу) |
| `display-lg` | `2.441rem` (39px) | 700 | 1.1 | `-0.025em` | Bricolage Grotesque | Заголовки секций |
| `display-md` | `1.953rem` (31.2px) | 700 | 1.15 | `-0.02em` | Bricolage Grotesque | Заголовки карточек-метрик |
| `heading-lg` | `1.563rem` (25px) | 600 | 1.2 | `-0.015em` | Bricolage Grotesque | Подзаголовки страниц |
| `heading-md` | `1.25rem` (20px) | 600 | 1.3 | `-0.01em` | DM Sans | Заголовки карточек |
| `body-lg` | `1.125rem` (18px) | 400 | 1.6 | `0` | DM Sans | Важный body текст |
| `body` | `1rem` (16px) | 400 | 1.6 | `0` | DM Sans | Основной body текст |
| `body-sm` | `0.875rem` (14px) | 400 | 1.5 | `0.005em` | DM Sans | Подписи, мета-информация |

| `data` | `0.875rem` (14px) | 400 | 1.4 | `0` | JetBrains Mono | Числа в таблицах |

### Правила типографики

- Максимальная ширина текстового блока: `65ch`
- Body text НИКОГДА ниже 16px
- Заголовки: line-height 1.05–1.2 (плотно). Body: 1.5–1.6 (просторно)
- Не больше 3 весов одного шрифта на странице
- ALL CAPS — только для коротких лейблов (максимум 3 слова), НИКОГДА для body текста
- Числовые метрики (калории, вес, граммы) — ВСЕГДА JetBrains Mono для табулярного выравнивания

## 4. Component Stylings

## 5. Layout Principles

### Шкала отступов (spacing scale)

Базовый unit: `0.5rem` (8px). Все отступы кратны этому значению.

| Токен | Значение | Применение |
|-------|----------|------------|
| `space-1` | `0.25rem` (4px) | Минимальный зазор между инлайн-элементами |
| `space-2` | `0.5rem` (8px) | Внутри компонентов, между иконкой и текстом |
| `space-3` | `0.75rem` (12px) | Между полями формы |
| `space-4` | `1rem` (16px) | Padding карточек (мобильный), между элементами в списке |
| `space-6` | `1.5rem` (24px) | Padding карточек (desktop), между группами |
| `space-8` | `2rem` (32px) | Между секциями внутри страницы |
| `space-12` | `3rem` (48px) | Между крупными секциями |
| `space-16` | `4rem` (64px) | Верхний отступ страниц |
| `space-24` | `6rem` (96px) | Hero-секции |

### Сетка

- **Desktop (>1024px):** Sidebar (240px) + Content area. Content: 12-колоночный CSS Grid, max-width `1200px`, gap `1.5rem`
- **Tablet (768–1024px):** Sidebar скрывается. Top bar + content. Grid 8 колонок
- **Mobile (<768px):** Полная ширина. Grid 4 колонки. Bottom nav

### Принципы раскладки

- **Левое выравнивание по умолчанию.** Центрирование — только hero-секция и пустые состояния
- **Асимметрия приветствуется.** Dashboard: метрики-карточки 2/3 + сайд-панель 1/3
- **Негативное пространство — осознанное.** Между секциями — `space-12` минимум. Элементы не прижаты друг к другу
- **Z-паттерн** для лендинга. **F-паттерн** для dashboard и форм

## 6. Depth & Elevation

### Система теней (тёплые, не серые)

| Уровень | CSS | Применение |
|---------|-----|------------|
| `elevation-0` | Нет тени | Фоновые контейнеры, секции |
| `elevation-1` | `0 1px 3px oklch(22% 0.015 55 / 0.06), 0 4px 12px oklch(22% 0.015 55 / 0.04)` | Карточки, dropdown |
| `elevation-2` | `0 2px 6px oklch(22% 0.015 55 / 0.08), 0 12px 32px oklch(22% 0.015 55 / 0.06)` | Модальные окна, поповеры |
| `elevation-3` | `0 4px 12px oklch(22% 0.015 55 / 0.1), 0 20px 48px oklch(22% 0.015 55 / 0.08)` | Sticky-хедер при скролле, toast-уведомления |

### Правила

- Тени ВСЕГДА тёплые (подкрашены к Espresso). Никогда чисто серые
- Не больше 3 уровней тени на одном экране
- Никакого `box-shadow` с `glow` (цветные тени-свечения) — это AI slop
- Бордеры предпочтительнее теней для разделения на одном уровне

## 7. Do's and Don'ts

### DO (делай)

- Используй Bricolage Grotesque для заголовков — это формирует характер бренда
- Числа-метрики — крупно, JetBrains Mono, с пониманием что это ГЛАВНАЯ информация
- Оставляй щедрое негативное пространство между секциями
- Тонируй нейтрали к тёплому (hue 55) — создаёт подсознательную связность
- Анимации — subtle, 150–200ms, `ease-out`. Только функциональные (появление, уход)
- Прогресс-бары — скруглённые, Sage для заполнения, Sand для фона
- Пустые состояния — тёплый текст + мягкая иллюстрация + CTA
- `oklch()` для всех цветов в CSS
- `rem` для всех размеров шрифтов
- `ch` для ограничения ширины текста (`max-width: 65ch`)
- Темы light/dark — через CSS custom properties + `prefers-color-scheme`

### DON'T (не делай)

- **Шрифты:** НЕ Inter, Roboto, Open Sans, Montserrat, Lato, Arial, system-ui для основного UI
- **Цвета:** НЕ фиолетовые градиенты, НЕ cyan-on-dark, НЕ неоновые акценты, НЕ чистый #000 или #fff
- **Эффекты:** НЕ glassmorphism, НЕ gradient text, НЕ glowing box-shadows, НЕ bounce-анимации
- **Лейаут:** НЕ всё центрировать. НЕ карточки внутри карточек. НЕ одинаковые отступы везде
- **Типографика:** НЕ ALL CAPS для body текста. НЕ размеры ближе чем 1.25x друг к другу. НЕ body меньше 16px
- **Паттерны:** НЕ «hero metric layout» (число + лейбл + 3 стата). НЕ monospace «для технического вайба»
- **Иконки:** НЕ emoji вместо иконок в навигации (только в контенте). Использовать Lucide Icons или Phosphor Icons
- **Бордеры:** НЕ толстый акцентный бордер на скруглённой карточке — clashes с border-radius

## 8. Responsive Behavior

### Breakpoints

| Имя | Ширина | Поведение |
|-----|--------|-----------|
| `mobile` | < 768px | 1 колонка. Bottom nav (5 пунктов). Карточки — full width. Метрики — 2 в ряд |
| `tablet` | 768px – 1024px | 2 колонки. Top nav + бургер. Sidebar скрыт. Метрики — 3 в ряд |
| `desktop` | 1024px – 1440px | Sidebar + content area. Метрики — 4 в ряд. Таблицы — полные |
| `wide` | > 1440px | Max-width 1440px с центрированием. Увеличенные отступы |

### Touch Targets

- Минимальный размер нажимаемого элемента: `44×44px` (WCAG)
- Расстояние между интерактивными элементами на мобильном: минимум `8px`

### Стратегия адаптации

- **Dashboard:** Метрики-карточки перестраиваются из 4-колоночного grid в 2-колоночный (mobile)
- **Таблицы:** На мобильном — карточный вид (каждая строка = карточка) вместо горизонтальной прокрутки
- **Формы:** Полная ширина на мобильном. Лейблы сверху (не сбоку)
- **Sidebar:** Скрывается в slide-over panel на мобильном
- **Графики/диаграммы:** Упрощённая версия на мобильном (скрыть легенду, уменьшить точки данных)

## 12. Анимации и Motion Design

### Принципы

- Анимации — функциональные, не декоративные
- Длительность: 150–300ms максимум
- Easing: `ease-out` для появления, `ease-in` для ухода. НЕ `bounce`, НЕ `elastic`
- Stagger для списков: 50ms между элементами

### Конкретные анимации

| Элемент | Анимация | Длительность | Easing |
|---------|----------|-------------|--------|
| Появление карточки | `opacity 0→1, translateY 8px→0` | 200ms | ease-out |
| Hover карточки | `translateY 0→-2px, shadow elevation-1→2` | 150ms | ease-out |
| Переход между табами | `opacity + slideX` | 200ms | ease-out |
| Модальное окно | `opacity 0→1, scale 0.95→1` | 250ms | ease-out |
| Toast-уведомление | `slideY from top + opacity` | 300ms | ease-out |
| Прогресс-бар | `width 0→N%` | 600ms | ease-out |
| Кнопка нажатие | `scale 1→0.97→1` | 100ms | ease-in-out |

## 14. Accessibility (a11y)

- Контраст текст/фон: минимум 4.5:1 (WCAG AA) для body, 3:1 для крупного текста
- Все интерактивные элементы доступны с клавиатуры (Tab, Enter, Escape)
- Focus-кольцо: `3px solid Terracotta / 15%` — видимое, но не агрессивное
- Aria-labels на иконках-кнопках
- Screen reader: скрытые лейблы для визуальных элементов
- `prefers-reduced-motion` — отключить все анимации
- `prefers-color-scheme` — автоматическая тема
- Минимальный touch target: `44×44px`
